Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

# TODO: Import 'r2_score'
from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # TODO: Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_pred=y_predict)
    
    # Return the score
    return score

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer: The benefit is to keep data for finding out about how well the model generalizes. Data that the model has been trained on will most often have a better score than other data sampled from the population (overfitting). Usually, the question we want answered is "how much better", and we can get an idea from seeing how the model we've constructed performs on the test set.

So the error of a model can be hard to deduce (except by analytic means) from the same data it has trained on. We sacrifice increased error due to variance that comes from having a smaller sample size. Having a training and test set to compare gives evidence on how it decomposes into error due to bias and variance.

If the model is "overfitting", it there will be few sample sets that achieve a score as high as that performed by the training set. If our testing set scores much lower than our training set, then this might be the situation. On the otherhand, if the model is not complex enough, the bias will be high. That is, the expected value of the model will be much different than the true function. This implies both the training set and testing set will both score low.

Keep in mind, the results on a single training and testing might indicate one of the above situations, but not necessarily. For instance, a high score on the training and low score on the testing can also occur when when our training size is too small, but where our model complexity is the same as the underlying function.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: The k-fold cross-validation technique is where you partition your data into it k-sets, , and train your model k-1 times, each time using on of the k-1 sets of the partition as your validation set, and the rest of the data as your training data.

With grid search, we split the data into training and testing data. Then, we implement the k-fold cross-validation technique on the training set to choose a our hyperparameters. Finally, we train the model on our training data with the choosen hyperparameters to obtain our predictor, and use the test data to score it.

Using grid search without a cross-validated would mean (a) using the test set to choose our hyperparameters, or (b) partition our data into a training, validation, and test set, and using our validation set to choose parameters.

(a) Because we end up using our testing model for training as well, the model will then likely score better on our test set than on unseen data.

(b) set means that when we choose our parameters based on the validation set it could cause the model to overfit the data. This is because we might be looking at a broad set of parameters, but measuring on the same validation dataset over and over again, meaning that the success of a parameter might have more to do with the validation set than success score of the parameter.

By using k-fold cross-validation on the training data, we get to see the score over many different validation sets, the choice of parameter will depend less on the validation set chosen. Also, once we choose our parameter, all of the training data will be used to build a predictor.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor 
from sklearn.metrics import make_scorer 
from sklearn.grid_search import GridSearchCV 

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': list(range(1,11))}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, cv=cv_sets, scoring=scoring_fnc)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
